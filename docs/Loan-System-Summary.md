## 💡 核心理念

### 贷款制的本质

> **"自我验证 + 滚动发展"**

**杭州小伙模式：**
1. AI 项目贷款$99（iOS 开发者账号年费）
2. AI 自己赚回 3 个$99，才能开下一个账号
3. 95% 以上无人干预
4. 不买服务器、不买域名、只花$99
5. 5 个月 120 个 App
6. 90%+ 付费率

**OpenClaw 版本：**
1. 启动成本：0 元（用现有 API 余额）
2. 偿还目标：赚回 3 倍 API 成本
3. 再投资：开启下一个项目
4. 决策机制：Agent 自主决定
5. 规模化：10-20 个小工具 App

## 🤖 7 个 Agent 角色

### 1. PM Agent（项目经理）

**职责：** 项目筛选、立项决策、资源分配、进度追踪

**决策逻辑：**
```yaml
if 项目预期收益 > 3 倍成本:
  批准立项
elif 项目连续 7 天无收入:
  终止项目
elif 项目已赚回 3 倍成本:
  开启下一个项目
```

---

### 2. Research Agent（市场研究）

**职责：** 需求分析、竞品调研、关键词研究、定价建议

**交付物：**
- 需求文档
- 竞品分析报告
- 定价策略

---

### 3. Architecture Agent（架构设计）

**职责：** 技术选型、架构设计、接口定义、代码规范

**交付物：**
- 技术方案文档
- 目录结构
- 接口文档
- 代码模板

**决策原则：**
- 优先使用无服务器方案
- 优先使用免费服务
- 优先使用现成模板

---

### 4. Dev Agent（OpenCode 开发）

**职责：** 代码实现、单元测试、Bug 修复、代码优化

**交付物：**
- 可运行的代码
- 单元测试
- 构建产物

**约束：**
- 不买服务器
- 不买域名
- 使用免费服务
- 代码可复用

---

### 5. Test Agent（测试验收）

**职责：** 功能测试、性能测试、兼容性测试、用户体验

**决策逻辑：**
```yaml
if 严重 Bug 数量 > 0:
  退回修复
elif 功能完整度 < 90%:
  退回补充
else:
  通过验收
```

---

### 6. Deploy Agent（发布部署）

**职责：** 打包发布、商店提交、ASO 优化、上架追踪

**交付物：**
- 已上架 App
- 下载数据
- 收入数据

---

### 7. Analytics Agent（数据分析）

**职责：** 收入追踪、成本核算、ROI 计算、项目评估

**决策逻辑：**
```yaml
if 收入 >= 3 倍成本:
  建议继续
elif 7 天收入 < 成本/10:
  建议停止
else:
  建议观察
```

## 📋 完整工作流（7 阶段）

### 阶段 1：需求发现（Research Agent）

```
1. 扫描应用商店排行榜
2. 分析用户需求（评论关键词、差评、好评）
3. 竞品分析
4. 输出需求文档

交付物：docs/projects/[项目名]/01-Requirements.md
```

---

### 阶段 2：立项决策（PM Agent）

```
1. 评估项目可行性
2. 决策：if 预期收益 >= 3 倍成本 → 批准
3. 分配预算（API 预算：¥50，时间预算：3 天）

交付物：docs/projects/[项目名]/02-Project-Charter.md
```

---

```
1. 技术选型（Flutter/React Native/Web）

交付物：docs/projects/[项目名]/03-Architecture.md

---

## 🎯 自动化决策机制

### 项目筛选标准

**准入条件：**
- 开发成本 <= ¥50
- 预期收入 >= ¥150 (3 倍)
- 开发周期 <= 3 天
- 无需服务器
- 可复用现有代码

**拒绝条件：**
- 需要后端服务器
- 需要购买域名
- 开发周期 > 7 天
- 市场竞争激烈
- 法律风险高

### 项目终止标准

**终止条件：**
- 连续 7 天无收入
- 收入 < 成本/10 (7 天)
- 严重 Bug 无法修复
- 审核被拒 3 次
- 法律风险出现

```

- 不买服务器（用免费层）
- 不买域名（用子域名）

- GitHub Pages (免费)

## 💡 最佳实践总结

### 1. 讨论时机

**应该用 Google AI 讨论的时机：**
- ✅ 项目立项时
- ✅ 技术选型时
- ✅ 方案决策时
- ✅ 风险评估时
- ✅ 遇到瓶颈时
- ✅ 需要创意时

**不应该用 Google AI 的时机：**
- ❌ 需要执行代码时
- ❌ 需要访问本地数据时
- ❌ 需要测试部署时
- ❌ 紧急 Bug 修复时

---

### 2. 讨论深度

**浅层讨论（5-10 分钟）：**
```
- 快速验证想法
- 获取初步建议
- 确认方向
```

**中层讨论（30-60 分钟）：**
```
- 可行性分析
- 风险评估
- 方案对比
```

**深层讨论（2-3 小时）：**
```
- 完整项目规划
- 技术架构设计
- 商业模式设计
- 详细时间线
```

---

### 3. 讨论技巧

**好的提示词：**
```
✅ 提供充分背景信息
✅ 明确讨论目标
✅ 提出具体问题
✅ 请求详细分析
✅ 要求提供案例
```

**不好的提示词：**
```
❌ "帮我做个项目"（太模糊）
❌ "这个怎么样"（缺少背景）
❌ "给个方案"（缺少细节）
```

---

### 4. 结果处理

**讨论后的行动：**
```
1. 保存讨论记录
2. 提取关键结论
3. 验证重要信息
4. 纳入项目文档
5. 执行可行建议
```

**不应该：**
```
❌ 盲目相信所有建议
❌ 不验证直接使用
❌ 忽略相反意见
❌ 只保存不执行
```

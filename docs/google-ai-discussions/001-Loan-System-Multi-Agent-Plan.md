## 一、核心理念

### 贷款制的本质

```
贷款制不是"贷款"，而是"自我验证 + 滚动发展"

核心逻辑：
1. 最小启动成本（$99）
2. 项目自我偿还（赚回 3 倍）
3. 利润再投资（开启下一个项目）
4. 无人干预（AI 自主决策）
5. 规模化取胜（概率 > 完美）
```

### 应用到 Multi-Agent

```
OpenClaw 版本贷款制：

1. 启动成本：0 元（用现有 API 余额）
2. 偿还目标：赚回 3 倍 API 成本
3. 再投资：开启下一个项目
4. 决策机制：Agent 自主决定
5. 规模化：10-20 个小工具 App
```

```yaml

  - 项目立项批准
  - 预算分配
  - 项目终止决定

输入：
  - Research Agent 的需求报告
  - Analytics Agent 的财务报表

输出：
  - 项目立项文档
  - 预算批准
  - 项目状态更新

决策逻辑：
  if 项目预期收益 > 3 倍成本:
    批准立项
  elif 项目连续 7 天无收入:
    终止项目
  elif 项目已赚回 3 倍成本:
    开启下一个项目
```

## 三、完整工作流

### 阶段 1：需求发现（Research Agent）

```
1. 扫描应用商店排行榜
   - 免费榜 Top 100
   - 付费榜 Top 100
   - 飙升榜 Top 50

2. 分析用户需求
   - 评论关键词提取
   - 差评分析（痛点）
   - 好评分析（爽点）

3. 竞品分析
   - 功能对比
   - 定价对比
   - 下载量估算

4. 输出需求文档
   - 目标用户
   - 核心功能
   - 差异化点
   - 定价建议

交付物：docs/projects/[项目名]/01-Requirements.md
```

### 阶段 2：立项决策（PM Agent）

```
1. 评估项目可行性
   - 开发成本估算（API token）
   - 预期收入（基于竞品）
   - ROI 计算

2. 决策
   if 预期收益 >= 3 倍成本:
     批准立项 → 进入阶段 3
   else:
     拒绝项目 → 返回阶段 1

3. 分配预算
   - API 预算：¥50
   - 时间预算：3 天
   - 功能范围：MVP

交付物：docs/projects/[项目名]/02-Project-Charter.md
```

### 阶段 3：架构设计（Architecture Agent）

```
1. 技术选型
   - 前端：Flutter / React Native / 纯 Web
   - 后端：无服务器优先（Firebase/Supabase 免费层）
   - 存储：本地优先（SQLite/LocalStorage）

2. 架构设计
   - 目录结构
   - 模块划分
   - 接口定义

3. 代码模板
   - 基础模板复用
   - 组件库选择
   - 样式规范

交付物：docs/projects/[项目名]/03-Architecture.md
```

### 阶段 4：开发实现（Dev Agent - OpenCode）

```
1. 初始化项目
   - 创建目录结构
   - 安装依赖
   - 配置环境

2. 功能开发
   - 核心功能优先
   - 迭代开发
   - 单元测试

3. 代码审查
   - 代码质量检查
   - 性能检查
   - 安全检查

4. 构建
   - 生成构建产物
   - 打包
   - 签名

交付物：build/[项目名]/app-release.ipa/apk
```

### 阶段 5：测试验收（Test Agent）

```
1. 功能测试
   - 核心功能测试
   - 边界条件测试
   - 异常处理测试

2. 性能测试
   - 启动速度
   - 内存占用
   - 电池消耗

3. 兼容性测试
   - iOS/Android 版本
   - 屏幕尺寸
   - 设备型号

4. 验收决策
   if 严重 Bug == 0 and 功能完整度 >= 90%:
     通过 → 进入阶段 6
   else:
     退回 → 返回阶段 4

交付物：docs/projects/[项目名]/05-Test-Report.md
```

### 阶段 6：发布部署（Deploy Agent）

```
1. 商店准备
   - 应用图标
   - 截图
   - 描述
   - 关键词

2. 提交审核
   - App Store Connect
   - Google Play Console
   - 回答审核问题

3. ASO 优化
   - 标题优化
   - 关键词优化
   - 截图 A/B 测试

4. 上架追踪
   - 审核状态
   - 上架时间
   - 初始下载

交付物：docs/projects/[项目名]/06-Deployment-Report.md
```

### 阶段 7：数据追踪（Analytics Agent）

```
1. 收入追踪
   - 每日收入
   - 累计收入
   - 收入趋势

2. 成本核算
   - API 成本
   - 开发者账号成本
   - 其他成本

3. ROI 计算
   - 累计收益 / 累计成本
   - 回本周期
   - 利润率

4. 决策建议
   if 收入 >= 3 倍成本:
     建议：开启下一个项目
   elif 7 天收入 < 成本/10:
     建议：停止项目
   else:
     建议：继续观察

交付物：docs/projects/[项目名]/07-Analytics-Report.md
```

## 五、自动化决策机制

### 项目筛选标准

```yaml
准入条件:
  - 开发成本 <= ¥50
  - 预期收入 >= ¥150 (3 倍)
  - 开发周期 <= 3 天
  - 无需服务器
  - 可复用现有代码

拒绝条件:
  - 需要后端服务器
  - 需要购买域名
  - 开发周期 > 7 天
  - 市场竞争激烈
  - 法律风险高
```

### 项目终止标准

```yaml
终止条件:
  - 连续 7 天无收入
  - 收入 < 成本/10 (7 天)
  - 严重 Bug 无法修复
  - 审核被拒 3 次
  - 法律风险出现

继续条件:
  - 收入 >= 成本 (已回本)
  - 收入 >= 3 倍成本 (成功)
  - 增长趋势良好
  - 用户评价好
```

### 预算分配规则

```yaml
单项目预算:
  API 成本：¥50
  时间成本：3 天
  功能范围：MVP

滚动规则:
  if 项目收入 >= 3 倍成本:
    下一项目预算 = 当前预算 × 1.2
  elif 项目收入 >= 成本:
    下一项目预算 = 当前预算
  else:
    下一项目预算 = 当前预算 × 0.8

总预算上限:
  月度 API 预算：¥500
  单项目上限：¥100
```
